# Release notes — v0.1.0-alpha (2026-07-02)

## Summary

This is CompanionBench's first public-alpha release: an API-first, open-source benchmark for
evaluating human-like companion communication in LLMs — initiative relevance, intervention timing,
emotional attunement, preference adaptation, abstention, and safety boundaries — across realistic,
multi-turn conversations. The pipeline runs end-to-end offline against a deterministic mock model
and live against real providers (opt-in, budget-capped). This release ships an expanded 186-task
suite (150 public + 36 held-out), a completed live evaluation of the EMOTomo model set via
OpenRouter across both suites, full open-source project scaffolding, and a local-first
verification story (there is no GitHub Actions CI right now — see
[`../ci-disabled/`](../ci-disabled/)).

CompanionBench is model- and vendor-neutral. EMOTomo is one example **model set** used as a first
case study; OpenRouter is one example **provider adapter**. Neither is the benchmark's identity —
see [`../public_claims.md`](../public_claims.md).

## Installation

Requires **Python 3.12+** and [**uv**](https://docs.astral.sh/uv/).

```bash
git clone https://github.com/ErbolTakhirov/Benchmark.git
cd Benchmark
uv sync --all-extras
```

## Quickstart

Offline, no API keys needed:

```bash
uv run companion-bench validate manifests/smoke.yaml
uv run companion-bench run --manifest manifests/smoke.yaml --model mock/empathetic-v1 --out runs/smoke
uv run companion-bench score --run runs/smoke
uv run companion-bench report --run runs/smoke
```

Live, opt-in (needs a provider key in `.env` and `COMPANIONBENCH_LIVE=1`):

```bash
export OPENROUTER_API_KEY=...
COMPANIONBENCH_LIVE=1 companion-bench run -m manifests/smoke.yaml \
  --model-set configs/model_sets/your-set.yaml --out runs/live \
  --live --yes --max-cost-usd 1 --limit-tasks 2 --limit-models 2
companion-bench score --run runs/live
companion-bench report --run runs/live
```

Full quickstart, real-provider setup, and cost/retry/secret handling:
[`../../README.md`](../../README.md) and [`../live_and_cost.md`](../live_and_cost.md).

## Benchmark status

- **Task suite:** 150 public tasks + 36 held-out tasks, 25/6 per family across six families
  (initiative, empathy, timing, adaptation, abstention, safety).
- **Scoring:** rule-based, deterministic, fully transparent. No LLM-as-judge or human calibration
  yet — both are planned (see [Roadmap](#roadmap-toward-v01-stable)).
- **Providers:** OpenAI, Anthropic, OpenRouter, Hugging Face Inference Providers, and a generic
  OpenAI-compatible adapter are interface-complete and have been exercised against real providers;
  local open-weight models (vLLM/Ollama) are a secondary track, not the MVP default.
- **CI:** no GitHub Actions right now (account-level billing lock, unrelated to this repo's code) —
  local verification is the source of truth: [`../local_verification.md`](../local_verification.md).

## Result caveats

This release ships a completed live evaluation: the EMOTomo model set (10 models via OpenRouter)
across the full 150-task public suite and the 36-task held-out split, 5 repeats each. Full numbers:
[`../results_v0_1_alpha.md`](../results_v0_1_alpha.md). Read these results conservatively:

- They are **companion-communication scores on this task suite**, not a general capability or
  safety ranking, and not evidence any model is "the most human."
- Scores from models whose confidence intervals overlap are **statistical ties**, not a fine
  ranking — trust large gaps, distrust close calls.
- The public-vs-held-out correlation (Pearson 0.968, Spearman 0.939) shows the suite generalizes
  well *for this task suite and these model versions*; it is not a guarantee that holds for future
  suite revisions or model updates.
- Cost, latency, and parse-rate figures depend entirely on what each provider reported at run time
  (2026-07-01/02) and will drift as providers change pricing, models, or infrastructure.

This is a **snapshot, not a final leaderboard** — see [`../public_claims.md`](../public_claims.md)
for the exact language this project holds itself to.

## Known limitations

- Rule-based scoring only — gameable by substring/keyword overlap; no LLM-as-judge or human gold
  calibration yet.
- A model that fails to return the expected structured envelope is penalized for format
  non-compliance, which is a separate failure mode from companion-communication quality (check
  parse rate before reading a low score as "bad at companionship").
- Tasks are synthetic, AI-authored, and English-only; not yet reviewed by outside human evaluators.
- The task suite is one generation at its current size — not yet stress-tested across many suite
  revisions.
- The default model is a deterministic mock; mock scores validate the pipeline, not model quality.
- Cost/token accounting is `null` when a provider doesn't report it — never invented.
- No GitHub Actions CI right now.

Full list: [`../../README.md#limitations`](../../README.md#limitations) and the
[benchmark card](../benchmark_card.md).

## Roadmap toward v0.1 stable

1. Start the LLM-as-judge path behind the existing `RubricEvaluator` interface, with a small human
   gold-label set for rules-vs-judge-vs-human calibration.
2. Reduce substring-gaming: paraphrase-robust matching (synonym sets, embeddings, or judge
   calibration) and an empathy verbosity/genericness penalty.
3. Get the task suite in front of outside human reviewers; add non-English coverage.
4. Layer safety detection (patterns + a classifier) so novel unsafe phrasings are caught, not just
   pre-authored patterns.
5. Stratified/blocked bootstrap by task, with per-task variance reported alongside the aggregate.
6. Add a provenance block to every report (provider, model-version snapshot, pricing `as_of`,
   scoring version) for defensible cross-run comparison.
7. Resolve the GitHub Actions billing lock (or adopt an alternative CI provider) and re-enable
   automated verification on every PR.
